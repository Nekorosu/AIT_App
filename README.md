# **Основные правила**
- [ ] — Никто не коммитит напрямую в `main` — только через Pull Request
- [ ] — Именование веток: `feature/название`, `fix/название`, `ui/название`
- [ ] — Один файл редактирует один человек — договариваться заранее
- [ ] — Дизайн и код согласовывают `x:Name` контролов до начала работы над окном
- [ ] — `config.json` никогда не коммитится — только `config.example.json`
- [ ] — Тестирование тестирует `main` после каждого merge, не личные ветки
